# 订阅保号通知系统 — 整体技术方案

> 目标：自托管的订阅/续费管理系统，通过 Telegram 提醒用户及时续费保号，支持多用户、多货币、多语言、多主题。

---

## 1. 总体架构

```
┌─────────────────────────────────────────────────────────┐
│                     浏览器 (Web UI)                       │
│   多语言(中/英/俄) · 多主题 · 仪表盘 · 日历 · 报表        │
└───────────────────────────┬─────────────────────────────┘
                            │ HTTPS / REST API
┌───────────────────────────▼─────────────────────────────┐
│                      后端服务 (App)                       │
│  鉴权(JWT) · 多用户 · 订阅CRUD · 报表统计 · 货币换算      │
├──────────────┬───────────────┬──────────────┬────────────┤
│  定时任务     │  Telegram     │  汇率拉取      │  图标存储   │
│ (到期扫描)    │  Bot 推送      │ (汇率API)     │ (本地/对象) │
└──────┬───────┴───────┬───────┴──────┬───────┴────────────┘
       │               │              │
   ┌───▼───┐      ┌────▼────┐    ┌────▼─────┐

部署：Linux（Docker Compose 一键部署，含 App + 数据库 + 反向代理）。

| 层 | 选型 | 理由 |
|----|------|------|

## 3. 需要对接的外部 API

### 3.1 Telegram Bot API（核心）
- **创建 Bot**：通过 @BotFather 获取 `bot_token`。
- **获取 chat_id**：用户需向 Bot 发送一次消息，或通过 `getUpdates` / 用户在网页绑定。
- 关键接口：
  - `GET  https://api.telegram.org/bot<token>/getMe` — 验证 token 是否有效。
  - `POST https://api.telegram.org/bot<token>/sendMessage` — 发送续费提醒。
    - 参数：`chat_id`, `text`（支持 Markdown/HTML）, `parse_mode`。
  - `GET  .../getUpdates` — 辅助获取用户 chat_id（绑定流程）。
- 提醒文案示例：
  > 🔔 *续费提醒*：你的「ChatGPT Plus」将于 *2026-06-25* 到期（还剩 7 天），金额 **$20.00 USD**。请及时续费保号。

### 3.2 汇率 API（实时汇率）
候选（按免费额度/稳定性排序）：
| 服务 | 免费额度 | 备注 |
|------|---------|------|
| **exchangerate.host** | 免费，无需 key（部分功能需 key） | 简单易用 |
| **open.er-api.com** | 免费，无需 key | `https://open.er-api.com/v6/latest/USD` |
| **exchangerate-api.com** | 免费需注册 key | 稳定 |
| **fixer.io / currencyapi** | 免费有限额 | 备选 |

策略：每日定时拉取一次基准汇率（如以 USD 为基），缓存入库；换算在后端完成。支持用户自定义货币与手动汇率覆盖。

## 4. 数据库模型（核心表）

```
users            (id, username, email, password_hash, locale, theme,
                  base_currency, telegram_chat_id, created_at)

categories       (id, user_id?, name, icon, is_system, sort)   # 系统预置 + 用户自定义
                 # 预置: 流媒体/AI/游戏/VPS/软件/会员/域名...

payment_methods  (id, user_id?, name, icon, is_system)
                 # 信用卡/Apple Pay/支付宝/微信/PayPal/银行卡...

currencies       (code, name, symbol, is_custom, user_id?)     # USD/CNY/EUR/RUB/...
exchange_rates   (base, quote, rate, updated_at)               # 实时汇率缓存

subscriptions    (id, user_id, name, icon, category_id, payment_method_id,
                  amount, currency, billing_type,               # recurring | one_time(买断)
                  cycle, cycle_count,                           # day/week/month/year + N
                  start_date, next_renewal_date, end_date?,
                  is_active, auto_renew, url, notes)

reminders        (id, subscription_id, days_before, channel)    # 提前N天, telegram
notification_log (id, subscription_id, sent_at, status, message)
```

要点：
- `billing_type` 区分**周期订阅**与**一次性买断（永久购买）**。
- `next_renewal_date` 由周期推算，续费后自动滚动到下一周期。

## 5. 核心功能模块

1. **鉴权与多用户**：注册/登录、JWT、每用户数据隔离、（可选）管理员角色。
2. **仪表盘**：当月支出、年度支出、即将到期、最近订阅 —— 全部按用户基准货币换算。
3. **订阅管理**：CRUD、周期/买断、自定义图标上传、分类与付款方式选择。
4. **分类管理**：系统预置全球主流分类（流媒体/AI/游戏/VPS…）+ 用户自定义分组。
5. **多货币**：全球常用货币内置 + 自定义货币，实时汇率，界面显示当前货币。
6. **多语言**：中 / 英 / 俄（vue-i18n）。
7. **多主题**：浅色/深色 + 多套配色，用户偏好持久化。
8. **视图**：列表视图 + **日历视图**（按到期日渲染）。
9. **报表分析**：支出洞察（趋势）、支出排行（按分类/订阅）、永久购买清单、即将续费。
10. **通知**：每日定时扫描 → 命中提醒规则 → Telegram 推送 → 写入日志。

## 6. 提醒调度逻辑

```
每天 09:00 (用户时区) 定时任务:
  for 每个 active 周期订阅:
     for 每条 reminder 规则 (提前 N 天):
        if next_renewal_date - today == N 且 今天未发过:
           组装文案 → Telegram sendMessage → 记录 notification_log
  续费完成后: next_renewal_date 滚动 += cycle
```

## 7. 部署方案（Linux）

```
docker-compose.yml
 ├─ app        (FastAPI + 前端静态文件)
 ├─ db         (PostgreSQL, 数据卷持久化)
 ├─ caddy/nginx(反代 + 自动HTTPS)
 └─ (可选) redis
```

环境变量：`DATABASE_URL`、`JWT_SECRET`、`TELEGRAM_BOT_TOKEN`、`EXCHANGE_API_KEY`、`TZ`。

一键启动：`docker compose up -d`。
